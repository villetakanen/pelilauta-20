# Feature: Cyan Living Style Book (app/cyan-ds)

## Blueprint

### Context
The Living Style Book (`app/cyan-ds`) is the source of truth for the Cyan Design System. It provides a sterile environment to develop, document, and verify components and tokens in isolation before they are integrated into the main RPG platform (`app/pelilauta`). It serves as the primary host for visual regression and E2E behavioral tests.

### Architecture
- **Framework:** Astro (SSR-first).
- **Core Architecture:** **Astro Content Collections** for all "books" (documentation entries), separating them from structural pages.
- **Registries (Collections):** Documentation is strictly categorized into four groups:
    - `principles`: Core identity and branding foundations (Chroma, Semantic Colors, Thematic Identity).
    - `styles`: CSS and structural tokens (Grid, Units, Z-index, Elevation).
    - `components`: Core Design System components (shared by all apps).
    - `addons`: Specialized micro-frontend / sub-site components (MFE-only).
- **Routing:** All dynamic pages use a mandatory collection-prefixed route: `/[collection]/[slug]` (e.g., `/principles/chroma`, `/styles/units-and-grid`). This hierarchy ensures zero slug collisions and reflects the data organization.
- **Layouts:** 
    - `app/cyan-ds/src/layouts/Book.astro`: The root shell that dynamically builds the navigation tray by querying each collection and sorting by their categories.

### Data Models
- **Book Frontmatter Schema (Zod):**
    ```typescript
    {
      title: z.string(), // Page header
      description: z.string().optional(),
      order: z.number().optional(), // Sorting priority within its category
      status: z.enum(['stable', 'alpha', 'draft']).default('stable')
    }
    ```

### Dependencies
- **Depends on:** `packages/cyan` (Components/Tokens), `packages/pelilauta-icons` (Tier 1 Assets).
- **Consumer:** Playwright E2E tests target dynamic routes generated by content collections.

### Anti-Patterns
- **No Hardcoded Navigation:** Do not manually add links to the `Book.astro` navigation tray. It must derive from `getCollection` and follow the sorting logic: `[Ordered by priority] -> [Alphabetical Fallback]`.
- **No Direct Pages for Books:** Documentation entries must live in `src/content/[coll]/` rather than `src/pages/`.

## Contract

### Definition of Done
- [ ] Documentation entries are stored in the correct `src/content/` subfolder.
- [ ] `src/content/config.ts` enforces the mandatory `title` and optional `order`.
- [ ] `Book.astro` navigation tray is auto-generated and sorted by the four mandatory groups: **Principles**, **Styles**, **Components**, and **Addons**.
- [ ] Inside each group, entries with an `order` appear first, followed by unordered entries sorted alphabetically by `title`.
- [ ] Dynamic routing correctly renders pages at `/[coll]/[slug]`.

### Regression Guardrails
- **Schema Safety:** Build fails if any documentation entry violates the mandatory frontmatter contract.
- **Slug Collisions:** Unique slugs are enforced per category.

### Testing Scenarios

#### Scenario: Documentation Navigation
```gherkin
Given the Cyan Living Style Book is running
When a user visits the root page
Then they can see links to Foundations (Principles) and Components
  And clicking a component link (e.g., "Tray") navigates to its documentation
```
- **Playwright E2E Test:** `app/cyan-ds/e2e/ds.spec.ts`

#### Scenario: Visual Regression Host
```gherkin
Given a component has a documented demo page
When visual regression tests are executed via Playwright
Then the demos on the page are evaluated for alignment, color, and state
  And the results match established snapshots
```
- **Playwright E2E Test:** `app/cyan-ds/e2e/components/[component].spec.ts`
