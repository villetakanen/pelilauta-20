Semantic colors bridge the gap between raw **Chroma** (HSL values) and functional UI roles. Instead of referencing `--chroma-primary-10`, components reference `--color-primary`.

## Principles: Roles

- **Primary** — brand identity, interactive elements (buttons, links, FABs). The teal→yellow hue shift is unique to Cyan.
- **Surface** — backgrounds, cards, sheets, overlays. Use lower steps (0–30) for dark themes, higher steps (70–100) for light themes.
- **On-[Token]** — Always provides accessible contrast for text or icons placed on that background.

## Primary Roles

| Token | Role |
|-------|------|
| `--color-primary` | Main brand color |
| `--color-on-primary` | Content on primary |
| `--color-primary-variant` | Subtle variant |

## Background

The app background sits below all surfaces. It is subtly tinted to distinguish from pure surface.

| Token | Role |
|-------|------|
| `--color-background` | App shell background |
| `--color-on-background` | Text on background |

## Text

Text tokens define a hierarchy from maximum contrast to de-emphasized.

| Token | Role |
|-------|------|
| `--color-text` | Default body text |
| `--color-text-high` | Maximum contrast (black/white) |
| `--color-text-low" | De-emphasized text |
| `--color-heading-1` | H1 headings |
| `--color-heading-2` | H2 headings |

## Theme Switching

All `--color-*` tokens use `light-dark()` and respond to the `color-scheme` property on `:root`. The semantic CSS sets `color-scheme: light dark`, so the browser's preferred color scheme applies automatically.
