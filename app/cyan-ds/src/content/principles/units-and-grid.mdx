Cyan follows a deterministic **8px baseline grid**. This creates a predictable vertical rhythm and horizontal alignment across all applications.

## 1. The Atomic Unit (8px)

The entire system is derived from a single CSS variable: `--cn-grid`. By anchoring this to `0.5rem` (8px at the standard 16px root), the entire UI remains rescalable and accessible.

| Token | Derivation | Computed (16px root) | Role |
| :--- | :--- | :--- | :--- |
| `--cn-grid` | `0.5rem` | **8px** | The atomic unit of alignment. |
| `--cn-gap` | `grid × 2` | **16px** | Standard element spacing (gutters). |
| `--cn-line` | `grid × 3` | **24px** | Standard row height / vertical rhythm unit. |

## 3. Border Radii

Cyan utilizes a soft, grid-aligned radius system to maintain a professional, slightly rounded aesthetic.

| Token | Multiplier | Computed |
| :--- | :--- | :--- |
| `--cn-border-radius-small` | `× 0.5` | **4px** |
| `--cn-border-radius-medium` | `× 1` | **8px** |
| `--cn-border-radius-large` | `× 2` | **16px** |
| `--cn-border-radius-xl` | `× 4` | **32px** |

---

## 4. Navigation & Rail Geometry

Specific tokens define the persistent structural units of the Cyan application shell.

- **Rail Width:** `var(--cn-grid) * 10` (**80px**). 
- **Tray Width:** `var(--cn-grid) * 42` (**336px**).

These units ensure that sidebar navigation is neither too cramped for icons nor too wide for primary content.
