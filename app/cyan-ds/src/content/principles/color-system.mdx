Cyan utilizes a **Layered Color Model** that separates brand identity from functional intent. This architecture ensures that specialized campaign or sub-site branding can be applied instantly without modifying component logic.

## 1. The 3-Tier Layering Model

1.  **The Chroma Layer (Mechanical):** 13-step tonal scales (MD3) where "raw" HSL values live. This layer is designed to be **Swappable**.
2.  **The Semantic Layer (Functional):** Maps Chroma steps to functional roles (e.g., `--cn-text`, `--cn-button-accent`). This layer is **Immutable**.
3.  **The Component Layer (Consumptive):** Components consume only semantic tokens. They never reference raw HSL values.

## 2. The Chroma Scales (The Ink)

Cyan provides the following foundational palettes. Each follows a 13-point tonal scale from 0 (Black) to 100 (White).

### Brand & Surface
- **Primary:** Rotates hue from **Teal (185°)** at step 0 to **Yellow (65°)** at step 100.
- **Surface:** Parameterized Navy slate (Hue 204°, Saturation 98%) for all UI containers.

Used for status, warnings, and localized feedback.

| Palette | Hue | Role |

| **Love** | 3° | Positive intent, heart/favorite actions. |

## 4. Semantic Immutability

> [!WARNING]
> While Chroma can be replaced, **Semantic Tokens must never be overridden**. 
> 
> You should never do: `button { --cn-button-accent: red; }`. 
> Instead, if you need a red button, you either use a dedicated functional class or, if it's for branding, you override the underlying Chroma token: `:root { --chroma-primary-50: red; }`.
