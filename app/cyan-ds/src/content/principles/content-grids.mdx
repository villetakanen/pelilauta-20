<section class="cn-content-prose">
  Content Grids are the structural backbone of Cyan. They prioritize readability and deterministic spacing by providing three distinct layout patterns — **Prose**, **Golden**, and **Triad** — all driven by container queries and golden-ratio proportions.

  ## 1. Structural Logic

  All content regions respond to the width of the parent `<main>` container using **Container Queries**.

  - **Mobile (< 60rem):** Gutter padding of `--cn-gap` (16px) is enforced, and layouts collapse to a single column.
  - **Desktop (>= 60rem):** Layouts expand into their respective proportions.

  ## 2. Layout Variants

  ### Prose (Readability Centered)
  The default single-column layout centered at **`67ch`**. Use `.cn-content-prose`.
</section>

<section class="cn-content-prose" style="background: var(--cn-surface-2); border-radius: var(--cn-border-radius-small);">
  <h4 style="margin-top: 0;">Centered Prose</h4>
  <p>This section is automatically centered to exactly 67ch on wide screens. The elastic gutters on either side absorb the remaining width of the `.cn-app-main` container.</p>
</section>

<section class="cn-content-prose">
  ### Golden (2.618fr : 1fr)
  A two-column layout using the golden ratio squared. Typical for thread views or main-detail views. Use `.cn-content-golden`. The demo below is a direct child of `.cn-app-main` (enabled by `multipart: true` in this page's frontmatter) so the container query fires at its true main-wide width.
</section>

<section class="cn-content-prose">
  ### Triad (medium + small + small)
  A three-column layout: one **medium** (`67ch / 1.618`) and two **small** (`67ch / 2.618`) columns, with `--cn-gap` between them. Use `.cn-content-triad`. The block is sized to its content and centered in the container when it fits; otherwise the three children stack.
</section>

<section class="cn-content-prose">
  ## 3. The Full-Width Bleed

  When an element needs to break the constraints (e.g. for banners or full-width imagery), use **`.cn-grid-full`**.

  <div class="cn-grid-full" style="background: var(--cn-button-accent); padding: var(--cn-gap); color: white; text-align: center; border-radius: var(--cn-border-radius-small);">
    <b>I am .cn-grid-full</b> (Spans all columns of the enclosing grid)
  </div>

  Outside the bleed, prose remains centered at 67ch.

  ## 4. Vertical Rhythm

  Consecutive grid containers are automatically separated by **`2 * --cn-gap`** (32px) to maintain consistent density. The two demo sections below illustrate this default spacing.
</section>

<section class="cn-content-prose">
  ## 5. Robust Selection

  The grid system uses `nth-child` to reliably place children regardless of their HTML tag type (e.g. mixed `article` and `aside`).
</section>
