## Core Spacing

The Cyan Design System is built on an **8px baseline grid**. All spacing and sizing tokens derive from a single unit, `--cn-grid`, making the system rescalable by changing one value.

| Token | Derivation | Computed (16px root) |
|-------|-----------|---------------------|
| `--cn-grid` | `0.5rem` | 8px |
| `--cn-gap` | `grid × 2` | 16px |
| `--cn-line` | `grid × 3` | 24px |

## Responsive Sizing

Cyan v20 uses **container queries** instead of global media-query breakpoints. Components respond to their container's inline size, not the viewport.

- The legacy `--cn-breakpoint-*` tokens (620px, 960px, 1365px) are **retired**
- Layout containers declare `container-type: inline-size`
- Components use `@container` rules to adapt at locally meaningful thresholds
- Threshold values are component decisions, not global constants

```css
/* Container setup */
.layout-region {
  container-type: inline-size;
}

/* Component responds to its container */
@container (min-width: 620px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}
```

## Button Sizing

All buttons have a minimum physical touch area of **56×56 pixels**, even if their visual appearance is smaller. This ensures accessibility on touch devices.

| Token | Value | Role |
|-------|-------|------|
| `--cn-button-physical-size` | 56px | Minimum touch target |
| `--cn-button-size` | 38px | Standard visual size |
| `--cn-navigation-button-size` | 56px | Navigation button size |

> **Note:** Button demos use `--cyan-primary` as a temporary background. These will use `--chroma-primary-40` once the chroma token system is implemented.

## Border Radius

Border radius tokens provide consistent corner rounding, all derived from `--cn-grid`.

| Token | Multiplier | Computed |
|-------|-----------|---------|
| `--cn-border-radius-small` | `grid × 0.5` | 4px |
| `--cn-border-radius-medium` | `grid × 1` | 8px |
| `--cn-border-radius-large` | `grid × 2` | 16px |
| `--cn-border-radius-xl` | `grid × 4` | 32px |

## Icon Sizes

Icon size tokens follow a non-linear scale and are defined as absolute `rem` values (not grid-derived).

| Token | Computed |
|-------|---------|
| `--cn-icon-size-xsmall` | 16px |
| `--cn-icon-size-small` | 24px |
| `--cn-icon-size` | 36px |
| `--cn-icon-size-large` | 72px |
| `--cn-icon-size-xlarge` | 128px |

## Layout Dimensions

Layout tokens define the dimensions of structural navigation elements.

| Token | Derivation | Computed |
|-------|-----------|---------|
| `--cn-width-rail` | `grid × 10` | 80px |
| `--cn-height-rail` | `grid × 9` | 72px |
| `--cn-width-tray` | `grid × 42` | 336px |
