Every component that floats above the page — trays, modals, toasts — needs a z-index. Without shared tokens, components compete with magic numbers. The `--cn-z-*` tokens define a single, explicit stacking order for the whole system.

## Hierarchy

Six layers, ordered back-to-front:

| Token | Value | Component |
|---|---|---|
| `--cn-z-tray` | 20000 | Side navigation tray |
| `--cn-z-rail` | 30000 | Bottom navigation rail |
| `--cn-z-tray-button` | 40000 | Tray toggle button |
| `--cn-z-reply-dialog` | 41000 | Docked reply dialog |
| `--cn-z-modal` | 50000 | Modal dialogs and overlays |
| `--cn-z-snackbar` | 60000 | Toast notifications |

Values are intentionally high (20 000–60 000) to avoid collisions with third-party styles, and spaced in increments of 10 000 to leave room between layers.

## Usage

Reference the token closest to your component's role:

```css
.my-overlay {
  position: fixed;
  z-index: var(--cn-z-modal);
}
```

Z-index only takes effect on **positioned elements** (`position` must be `relative`, `absolute`, `fixed`, or `sticky`).

To add a new layer, insert it between existing tokens by choosing a value in the gap. Update this page and the token file together.
