Semantic tokens map [chroma palettes](/principles/chroma) to functional UI roles. Components reference `--color-*` tokens — never `--chroma-*` directly — so theme switching works automatically via `light-dark()`.

## Surfaces

Surface tokens define the background layers of the UI. Higher numbers indicate more visual elevation.

| Token | Role |
|-------|------|
| `--color-surface` | Default surface |
| `--color-surface-1` | Elevated surface (level 1) |
| `--color-surface-2` | Elevated surface (level 2) |
| `--color-surface-3` | Elevated surface (level 3) |
| `--color-surface-4` | Elevated surface (level 4) |
| `--color-on-surface` | Text/icons on surface |
| `--color-on-surface-secondary` | Lower-emphasis text |

## Text

Text tokens define a hierarchy from maximum contrast to de-emphasized.

| Token | Role |
|-------|------|

| `--color-text-low` | De-emphasized text |
| `--color-heading-1` | H1 headings |
| `--color-heading-2` | H2 headings |

## Theme Switching

All `--color-*` tokens use `light-dark()` and respond to the `color-scheme` property on `:root`. The semantic CSS sets `color-scheme: light dark`, so the browser's preferred color scheme applies automatically. Components never need to handle theme logic — they just reference `--color-*` tokens.
